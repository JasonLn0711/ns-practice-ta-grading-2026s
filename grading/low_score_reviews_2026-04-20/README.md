## 共用評分規範

- `HW6（程式）` 評分只看程式實作是否真的完成、是否正確、是否可驗證。
- `HW6（圖）` 評分只看圖像證據是否真正符合題目要求、是否完整、是否可回查。
- 程式端必要要求包含：
  - 架構替換。
  - 計算圖與程式標記一致性（graph-to-code notation alignment）。
  - 小批次隨機梯度搭配動量（mini-batch stochastic gradient with momentum）。
  - MNIST 測試準確率（test accuracy）。
  - 可稽核程式證據。
- 圖像端必要要求包含：
  - 計算圖（computational graph）。
  - 已學得濾波器（learned filters）。
  - 任意輸入數字的中間特徵圖（intermediate feature maps）。
  - 圖像標題、標籤、圖說與可回查性。
- 第一原理規則：
  - 沒有可驗證證據就不能給必要項目滿分。
  - 任何扣分都必須有明確書面理由。
  - 高準確率不能抵銷架構、計算圖、動量訓練或圖像證據缺失。
